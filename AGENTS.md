# Workspace instructions

Before editing this website, read `memory.md` in full. It is the authoritative
local record of the approved "Engineered Uptime" design direction, content
facts, architecture, known limitations and user preferences.

After any meaningful design, content, behavior or architecture change, update
`memory.md` before finishing. Record the decision, affected files and the
reasoning or constraint future chats should preserve. Do not add trivial edit
logs.

Work in the top-level `docs/` site unless the user explicitly requests the
nested duplicate. Preserve the existing static HTML/CSS/vanilla-JS structure.
Do not start a server, open a browser preview, render pages, or run tests unless
the user explicitly requests it.
